Methodology

This benchmark compares public perp order book execution quality for StandX, edgeX, GRVT, Lighter, Extended, and Nado on BTC, ETH, and SOL, with Hyperliquid and Aster shown as reference venues only. It is not a trading signal, liquidation monitor, whale tracker, vault dashboard, or venue marketing page.

Data Sources

  • Hyperliquid: POST https://api.hyperliquid.xyz/info with type=l2Book. Public response returns up to 20 levels per side.
  • StandX: GET https://perps.standx.com/api/query_depth_book for depth and slippage; GET https://perps.standx.com/api/query_symbol_market for quoted top-of-book spread. StandX SOL-USD is tracked as the SOL market when it appears in public symbol data.
  • Aster: GET https://fapi.asterdex.com/fapi/v1/depth for USDT-margined perpetual futures.
  • edgeX: GET https://pro.edgex.exchange/api/v1/public/quote/getDepth with public contract ids. The public REST snapshot supports fixed depth levels; this benchmark requests level 200.
  • GRVT: POST https://market-data.grvt.io/full/v1/book for public perpetual order book depth. This benchmark requests 50 levels per side.
  • Lighter: GET https://mainnet.zklighter.elliot.ai/api/v1/orderBookOrders for public order-level snapshots. This benchmark requests up to 250 orders per side and aggregates them into price levels before computing metrics.
  • Extended: GET https://api.starknet.extended.exchange/api/v1/info/markets/{market}/orderbook for public perpetual order book depth.
  • Nado: GET https://gateway.prod.nado.xyz/v1/query?type=market_liquidity for public perpetual market liquidity. This benchmark requests 50 levels per side and converts x18 price and size values into price levels.

Cadence

The collector runs every 30 to 60 seconds. The default is 60 seconds. Latest public metrics can be refreshed every collector round, while 7 day history is exported as 15 minute rollups every 5 minutes.

Spread

mid = (best_bid + best_ask) / 2

spread_bp = ((best_ask - best_bid) / mid) * 10000

Spread is a top-of-book signal and can be affected by venue tick size or public-book aggregation. Depth and estimated taker slippage are the primary comparability metrics.

3bp, 5bp, and 10bp Depth

Bid depth sums price * size where price >= best_bid * (1 - bp / 10000).

Ask depth sums price * size where price <= best_ask * (1 + bp / 10000).

The public table shows two-sided total depth: depth_total_usd = depth_bid_usd + depth_ask_usd. JSON and SQLite keep bid, ask, and total fields for each depth band.

100,000 and 1,000,000 USD Estimated Taker Slippage

A buy order consumes asks from best ask upward until the target notional is filled. A sell order consumes bids from best bid downward. The public table shows both 100,000 USD and 1,000,000 USD average taker slippage.

If the returned public book cannot fill the target notional on either side, that target-size metric is marked insufficient public depth and displayed as N/A.

buy_slippage_bp = ((buy_avg_px - mid) / mid) * 10000

sell_slippage_bp = ((mid - sell_avg_px) / mid) * 10000

Comparability Limits

  • Only public order book data is used.
  • Hidden, private, or venue-internal liquidity is not measured.
  • Hyperliquid public books are limited to 20 levels per side.
  • Hyperliquid and Aster are displayed as reference venues and are excluded from public Best calculations and daily winner summaries.
  • Tracked venue-market pairs are not replaced with alternate markets; if a public symbol disappears, that state is surfaced as N/A: not listed.
  • edgeX, GRVT, Lighter, Extended, and Nado are included as emerging benchmark venues under the same public-book method, not as endorsed or sponsored venues.

Public Data Files

  • data/latest.json: latest target list and latest comparable metric rows.
  • data/health.json: export time, latest sample age, expected target count, valid sample count, failed count, not-listed count, insufficient-depth count, unavailable count, and per venue/market recent status.
  • data/history-7d.json: 15 minute venue/market rollups from persistent SQLite history, including sample counts and insufficient-depth counts.
  • data/daily-summary.json: copyable daily market notes. These do not contain trading advice.
  • data/anomalies.json: public anomaly events with metric, venue, market, start/end time, baseline, observed value, message, and dedupe key when available.

The static JSON files are the public API for this milestone. CSV export and an interactive API are not currently provided. See docs/public-data.md in the repository for field semantics, freshness semantics, and consumer guidance.
